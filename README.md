## WHAT IS THIS BOT?

Think of the bot as a very smart assistant that watches the currency markets
24 hours a day, 7 days a week so you do not have to. It uses a professional
trading strategy called Smart Money Concepts (SMC) — the same way banks and
large institutions analyse markets — and combines it with machine learning to
score each potential setup.

When it finds a high-quality trade opportunity it sends an alert straight to
your Telegram. If you have connected your broker account it can also place the
trade automatically, manage the position, move the stop loss to protect your
profit, and close everything for you.

### What the bot does automatically

- Scans 12 currency pairs and commodities every 15 minutes around the clock
- Detects Break of Structure, Order Blocks, Breaker Blocks, and liquidity sweeps
- Filters out setups when major economic news is imminent
- Scores every setup with machine learning (XGBoost + Random Forest ensemble)
- Sends a detailed Telegram alert with entry, stop loss, and two take-profit levels
- Executes the trade on your broker account if auto-trading is enabled
- Closes 50% of the position at Take Profit 1
- Moves the stop loss to breakeven after TP1 is hit
- Closes the remaining 50% at Take Profit 2
- Sends you a Telegram notification at every stage
- Reconciles any trades that closed while the bot was offline

### Pairs monitored

EURUSD, GBPUSD, USDJPY, AUDUSD, USDCAD, NZDUSD, USDCHF, EURGBP, EURJPY,
GBPJPY, XAUUSD, XAGUSD

## SYSTEM ARCHITECTURE — THE BIG PICTURE

The system has three separate machines, each with a distinct job. You must
set up all three for the bot to work fully. Read this section carefully before
doing anything — it will save you a lot of confusion later.

```
+-------------------------------------------------------------+
|  MACHINE 1 — THE BRAIN                                      |
|  Linux cloud server                                         |
|  YOUR CHOICE: Oracle Cloud (recommended) OR AWS             |
|                                                             |
|  What runs here:                                            |
|    bot.py           handles all Telegram messages           |
|    scheduler.py     triggers market scans every 15 mins     |
|    smc_strategy.py  analyses charts for SMC setups          |
|    ml_models.py     scores setups with machine learning     |
|    database.py      reads and writes to Supabase            |
|    mt5_connector.py sends trade orders to Machine 2         |
|    and other parts of the bot.                              |
|                                                             |
|  Cost: FREE (Oracle Always Free tier or AWS Free tier)      |
+-------------------------------------------------------------+
         |
         | Sends trade instructions over the internet (HTTP)
         |
+-------------------------------------------------------------+
|  MACHINE 2 — THE HANDS                                      |
|  Windows VPS — always Windows, regardless of which          |
|  cloud provider you chose for Machine 1                     |
|                                                             |
|  What runs here:                                            |
|    mt5_worker.py    a small web server that receives        |
|                     trade instructions from Machine 1       |
|    MetaTrader 5     the broker software that actually       |
|                     places orders in your account           |
|                                                             |
|  WHY WINDOWS? MetaTrader 5 is a Windows-only program.      |
|  It cannot run on Linux. This machine is ALWAYS a Windows  |
|  VPS regardless of what you chose for the brain server.    |
|                                                             |
|  Cost: ~$6.50/month (Contabo, Vultr, AWS, or similar)           |
+-------------------------------------------------------------+
         |
         | Both machines read from and write to the database
         |
+-------------------------------------------------------------+
|  MACHINE 3 — THE MEMORY                                     |
|  Supabase (free cloud database — no server to manage)       |
|                                                             |
|  Stores: all users, risk settings, trade history, signals  |
|                                                             |
|  Cost: FREE                                                 |
+-------------------------------------------------------------+
```

   `5512345678:AAHabcdefGHIjklMNOpqrSTUvwxYZ12345`

8. Now get your own Telegram user ID (to make yourself the admin):
   — Search for `@userinfobot` in Telegram

### STEP 3 — GENERATE YOUR ENCRYPTION KEY

Every user's broker password is encrypted before being saved to the database.
The encryption key is the master secret — without it the stored passwords are
completely unreadable even if someone accessed the database directly.

Check if Python is on your computer:
- Windows: open Command Prompt (Windows key → type `cmd` → Enter), type `python --version`
- Mac: open Terminal (Cmd + Space → type Terminal → Enter), type `python3 --version`
- If you see `Python 3.x.x` you are ready
- If not, download Python 3.11 from [https://www.python.org/downloads](https://www.python.org/downloads)
  — during installation, check the box **"Add Python to PATH"**

Install the cryptography library if needed:
```
pip install cryptography
```

Generate the key:

Windows:
```cmd
python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"
```

Mac / Linux:
```bash
python3 -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"
```

You will see output like:
```
gAAAAABh5K8j9X2wZmNpQrStUvWxYz1234567890ABCDefghIJKLmnopqrstuv==
```

Copy the entire string and save in Notepad labelled: `ENCRYPTION_KEY=`

Back this up in a password manager immediately. If you lose it, all connected
broker accounts must be re-linked.

Do the same for `MT5_WORKER_API_KEY`

### STEP 7 — CONFIGURE AND START THE MT5 WORKER

Create the project folder:
1. Open File Explorer (`Windows + E`)
2. Go to `C:\` → right-click → New → Folder → name it `NixieTrades`

Copy `mt5_worker.py` from this repository into `C:\NixieTrades`.
Easiest method: upload to Google Drive from your local computer, then download
inside the VPS browser.

Create the configuration file:
1. Open Notepad
2. Paste and fill in all your values:

```
TELEGRAM_BOT_TOKEN=paste_your_bot_token_here
SUPABASE_URL=paste_your_supabase_url_here
SUPABASE_KEY=paste_your_supabase_key_here
ENCRYPTION_KEY=paste_your_encryption_key_here
MT5_WORKER_API_KEY=invent_a_strong_random_password_here
LOG_LEVEL=INFO
```

For `MT5_WORKER_API_KEY` — invent a strong password such as with the same way `ENCRYPTION_KEY` was generated
Save it in Notepad labelled: `MT5_WORKER_API_KEY=`

3. File → Save As → navigate to `C:\NixieTrades`
4. File name: `.env` (dot at the start)
5. Save as type: **All Files**
6. Click Save

Install required packages:
1. Open Command Prompt (`Windows + R`, type `cmd`)
2. Run:

```cmd
cd C:\NixTrades
pip install flask waitress MetaTrader5 python-dotenv requests cryptography
```

Test the worker runs:
```cmd
python mt5_worker.py
```
You should see: `MT5 Worker starting on port 8000...`
Press `Ctrl + C` to stop.

Set up auto-start on reboot:
1. Open Notepad and paste:
```batch
@echo off
cd C:\NixTrades
waitress-serve --host=0.0.0.0 --port=8000 --threads=16 mt5_worker:app
```
2. Save as `C:\NixTrades\start_worker.bat` (Save as type: All Files)
3. Press `Windows + R`, type `shell:startup`, press Enter
4. Right-click `start_worker.bat` → Create shortcut
5. Drag the shortcut into the Startup folder

The worker now starts automatically every time the VPS reboots.

##  WHAT HAPPENS AUTOMATICALLY

Once the bot is running and you are subscribed, here is everything it does
on its own without you needing to do anything:

**Every 15 minutes, all day, every day:**
- Scans EURUSD, GBPUSD, USDJPY, AUDUSD, USDCAD, NZDUSD, USDCHF, EURGBP,
  EURJPY, GBPJPY, XAUUSD, XAGUSD
- Checks the Daily chart for trend direction
- Skips ranging pairs (no clear direction)
- On trending pairs: looks for Double Break of Structure and Market Structure Shift on the H1 chart
- Finds the best unmitigated Order Block or Breaker Block (the anchor zone)
- Waits for an M15 liquidity sweep to confirm the setup
- Scores the setup with machine learning
- Skips the setup if major news is within 30 minutes
- If the score passes the threshold, sends you a detailed alert

**Alert message contains:**
- Symbol and direction (BUY or SELL)
- Setup type (e.g., Unicorn, BOS, MSS)
- Entry price, Stop Loss, Take Profit 1, Take Profit 2
- Risk-to-reward ratio
- Machine learning score
- Any upcoming news warning

**When a trade is open (if auto-trading enabled):**
- Monitors the position every 60 seconds
- When price reaches Take Profit 1: closes 50% of the position
- Immediately moves Stop Loss to breakeven (your entry price) +5pips for buffer
- Sends you a Telegram notification
- When price reaches Take Profit 2: closes the remaining 50%
- Records the result in the database

**Daily at 6:30 AM UTC:**
- Sends a morning briefing with market overview

**Daily at 8:00 AM UTC:**
- Sends a list of high-impact economic news for the day

**Every 30 minutes:**
- Reconciles open trades against MT5 to catch any trades that closed
  while the bot was offline or restarting

##  TROUBLESHOOTING

Below are the most common problems and how to fix each one.
For every problem, start by looking at the bot's log file — it usually tells
you exactly what went wrong.

**View the log file:**
```bash
cat ~/nixie-trades-bot/nixie_trades_bot.log
```
Or to watch it live as the bot runs:
```bash
tail -f ~/nixie-trades-bot/nixie_trades_bot.log
```
Press `Ctrl + C` to stop watching.

---

### Problem: The bot is not responding in Telegram

**What this looks like:**
You send a command and get no reply.

**What to check:**
1. Is the bot running?
   ```bash
   screen -ls
   ```
   You should see `nixie-bot` in the list. If not, the bot has stopped.

2. Start it again:
   ```bash
   screen -S nixie-bot
   cd ~/nixie-trades-bot
   source venv/bin/activate
   python bot.py
   ```

3. Check the log for errors:
   ```bash
   cat ~/nixie-trades-bot/nixie_trades_bot.log | tail -50
   ```

---

### Problem: SSL or database connection errors

**What this looks like in the log:**
```
ERROR | [SSL: SSLV3_ALERT_BAD_RECORD_MAC]
```
or
```
ERROR | supabase connection failed
```

**What this means:**
A momentary internet hiccup between your server and the Supabase database.
The bot has a retry system built in — it will try up to 4 times automatically.
If you see this error occasionally, it is normal and does not require action.
If you see it constantly, check your server's internet connection.

---

### Problem: `local variable 'mins' referenced before assignment`

**What this means:**
A known bug in scheduler.py that has been fixed. Apply the fix from the
`HOTFIX_scheduler_mins_and_inducement.md` file.

---

### Problem: MT5 worker not reachable

**What this looks like in the log:**
```
ERROR | MT5 worker request failed: Connection refused
```

**What to check:**
1. Connect to your Windows VPS via Remote Desktop
2. Is the `start_worker.bat` window open and running?
   If the window is closed, double-click `start_worker.bat` to restart it
3. Check that the IP address in your Linux `.env` file matches your Windows VPS IP
4. Some VPS providers change the IP address on reboot — check this if it was
   working before

---

### Problem: No setup alerts are being sent

**What to check:**
1. Send `/status` to the bot — confirm you are subscribed
2. Look at the log file during a market scan:
   ```bash
   tail -f ~/nixie-trades-bot/nixie_trades_bot.log
   ```
   Wait 15 minutes and watch what happens during the scan
3. You may see `D1 trend is RANGING` for most pairs — this is normal during
   low-volatility periods. The bot only fires setups on trending pairs.
4. You may see `Skipping: news blackout` — this means a major economic event
   is imminent and the bot correctly refused to trade around it

---

### Problem: Trade execution fails (auto-trading)

**What this looks like:**
The alert arrives but no trade is placed, and the log shows an MT5 error.

**Common causes:**
- Your broker account does not have enough free margin
- The symbol name format is wrong for your broker
  (some brokers use `EURUSDm` instead of `EURUSD`)
- Your MT5 session expired and needs to be reconnected
  (send `/disconnect_mt5` then `/connect_mt5` again)
- The Windows VPS lost connection to the broker server

---

### Problem: Win rate showing 0% or not updating

This is fixed by the database reconciliation patch. Make sure you have
applied all patches from the hotfix documents. The `/status` command shows
win rate based on closed trades only. If you have no closed trades yet,
the win rate will show "no closed trades yet."
