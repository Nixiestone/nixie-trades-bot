## WHAT IS THIS BOT?

Think of the bot as a very smart assistant that watches the currency markets 24 hours a day,
7 days a week so you do not have to. It uses a professional trading strategy called Smart
Money Concepts (SMC) — the same way banks and large institutions analyse markets — and combines
it with machine learning to score each potential setup.

When it finds a high-quality trade opportunity it sends an alert straight to your Telegram.
If you have connected your broker account it can also place the trade automatically, manage
the position, move the stop loss to protect your profit, and close everything for you.

### What the bot does automatically

- Scans 12 currency pairs and commodities every 15 minutes around the clock
- Detects Break of Structure, Order Blocks, Breaker Blocks, and liquidity sweeps
- Filters out setups when major economic news is imminent
- Scores every setup with machine learning (XGBoost + Random Forest + GradientBoosting ensemble)
- Sends a detailed Telegram alert with entry, stop loss, and two take-profit levels
- Executes the trade on your broker account if auto-trading is enabled
- Closes 50% of the position at Take Profit 1
- Moves the stop loss to breakeven after TP1 is hit
- Closes the remaining 50% at Take Profit 2
- Sends you a Telegram notification at every stage
- Reconciles any trades that closed while the bot was offline

### Pairs monitored

EURUSD, GBPUSD, USDJPY, AUDUSD, USDCAD, NZDUSD, USDCHF, EURGBP, EURJPY,
GBPJPY, XAUUSD, XAGUSD

### STEP 4 — GENERATE YOUR ENCRYPTION KEY

Every user's broker password is encrypted before being saved to the database.
The encryption key is the master secret that locks and unlocks those passwords.
Without it, all stored broker passwords are permanently unreadable.

Check if Python is installed on your computer:

**On Windows:**
- Press the Windows key, type `cmd`, press Enter
- Type `python --version` and press Enter
- If you see `Python 3.x.x` you are ready

**On Mac:**
- Press `Cmd + Space`, type `Terminal`, press Enter
- Type `python3 --version` and press Enter
- If you see `Python 3.x.x` you are ready

**If Python is not installed:**
- Go to [https://www.python.org/downloads](https://www.python.org/downloads)
- Download Python 3.11
- During installation, check the box that says **Add Python to PATH**
- Complete the installation

Install the cryptography library:
```
pip install cryptography
```

Generate the encryption key:

**On Windows (in Command Prompt):**
```cmd
python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"
```

**On Mac or Linux (in Terminal):**
```bash
python3 -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"
```

You will see a long string like:
```
gAAAAABh5K8j9X2wZmNpQrStUvWxYz1234567890ABCDefghIJKLmnopqrstuv==
```

Copy the entire string and save it in Notepad labelled: `ENCRYPTION_KEY=`

Back this up in a password manager immediately. If you ever lose this key,
all broker accounts connected to the bot must be re-linked from scratch.

## PART 4 — CONNECT YOUR BROKER ACCOUNT VIA METAAPI

The bot is running. Now connect your MetaTrader 5 broker account so the bot
can place trades automatically.

### How this works:

When you connect your account, the bot registers it with MetaApi. MetaApi logs
in to your broker using the credentials you provide, and keeps a permanent
connection. From that point, every time the bot detects a setup and wants to
place a trade, it sends one instruction to MetaApi and MetaApi executes it.

The first time you connect an account, MetaApi needs 1 to 2 minutes to set
everything up. After that, it is instant every time.

## WHAT HAPPENS AUTOMATICALLY

Once the bot is running and you are subscribed, here is everything it does
without you needing to do anything:

**Every 15 minutes, all day, every day:**
- Scans all 12 monitored pairs on multiple timeframes
- Checks the Daily chart for the overall trend direction
- Skips pairs that are ranging with no clear direction
- For trending pairs, looks for Double Break of Structure or Market Structure Shift on H1
- Identifies the best unmitigated Order Block or Breaker Block
- Waits for a confirmed M15 liquidity sweep (price hunting stops before reversing)
- Scores the setup with three machine learning models
- Skips the setup if major economic news is within 30 minutes
- If the score passes the threshold, calculates entry, stop loss, and two take-profit levels
- Sends the alert to all subscribed users

**When an alert is sent:**
- The message shows: symbol, direction (BUY or SELL), entry price, stop loss, two take-profit levels, risk-reward ratios, machine learning score, and any news warnings
- If your MT5 account is connected, the trade is placed automatically

**When a trade is open:**
- Position is monitored every 10 seconds
- When price reaches Take Profit 1, you receive a confirmation asking whether to close 50% now or hold to TP2
- If you do not respond within 5 minutes, the full position continues to TP2 automatically
- After TP1 is confirmed, stop loss is moved to breakeven (entry price plus a small buffer)
- When price reaches Take Profit 2, the remaining position closes automatically
- You receive a notification at every stage

**If you enable autonomous management in settings:**
The TP1 partial close and breakeven move happen automatically without asking you first.
You are notified after each action. This is configured via `/settings`.

**Daily at 6:30 AM your local time:**
- Sends a market structure summary showing the trend on each major pair

**Daily at 8:00 AM your local time:**
- Sends a list of all high-impact economic news events for the day

**Every Sunday at 9:00 AM your local time:**
- Sends a weekly market overview and upcoming week news events

**Every 30 minutes:**
- Reconciles all open trades against MetaApi to catch any trades that closed
  while the bot was temporarily disconnected

Check if the bot is running:
```bash
screen -ls
```

```bash
screen -S nixie-bot
cd ~/nixie-trades-bot

python bot.py
```

---

### Problem: Database connection errors

```
ERROR | supabase connection failed
```
This is usually a momentary internet blip. The bot retries automatically up to 4 times.
If it persists, check your `SUPABASE_URL` and `SUPABASE_KEY` in the `.env` file.

---

### Problem: MetaApi account not connecting

```
ERROR | MetaApi account deployment failed
```

Things to check:
1. Is the `METAAPI_TOKEN` in your `.env` file correct? It must match exactly what MetaApi gave you.
2. Is the account number digits only? No spaces or letters.
3. Is the server name exactly as shown in MT5 Tools → Options → Server?
4. Does your broker allow API connections? Most do but some demo accounts restrict it.
5. First connection takes up to 2 minutes — wait before concluding it failed.

---

### Problem: No setup alerts are being sent

Check the log during a scan:
```bash
tail -f ~/nixie-trades-bot/nixie_trades_bot.log
```
Wait 15 minutes and watch what the scanner reports. Common messages you might see:

- `D1 trend is RANGING` — the pair has no clear direction so the bot correctly skips it
- `Skipping: news blackout` — a major economic event is imminent
- `AWAITING INDUCEMENT` — a setup is forming but the stop hunt has not happened yet
- `ML consensus below threshold` — the machine learning models are not confident enough

All of these are normal. The bot only fires alerts when all conditions align.

---

### Problem: Trade execution fails

If the alert arrives but no trade is placed:
1. Send `/status` and check if your account shows as connected
2. Send `/disconnect_mt5` then `/connect_mt5` to re-link the account
3. Check the log for a specific MetaApi error message
4. Verify your broker account has sufficient margin

---

### Problem: Win rate showing 0%

The `/status` command shows win rate from closed trades only. If you have no
closed trades yet, this is normal. The rate updates every time a trade closes.
